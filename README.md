# FUSE Test Repository for Data Science Notebooks

This repository is designed to test FUSE (Filesystem in Userspace) implementations, specifically for scenarios where a data science notebook needs to interact with local file dependencies, read data from relative paths, and write output to directories within the mounted filesystem.

The structure is intentionally designed with local Python dependencies (`dep1.py`, `dep2.py`) and data files (`data1.json`, `data2.json`) that the main notebook (`data_science_notebook.ipynb`) imports and reads using relative paths. This setup helps verify that FUSE mounts correctly expose the directory structure to the notebook environment and handle I/O operations as expected.

## Purpose

The primary goal of this repository is to simulate a realistic data science environment with local dependencies to test:
- **Relative path resolution:** Ensuring `./dep1.py` or `data1.json` are correctly resolved from within the notebook.
- **Local module imports:** Verifying that Python's `import` statement can find and load modules from `.py` files residing on the FUSE mount.
- **File I/O:** Reading from `.json` files and writing new `.json` files to an output directory.
- **Directory creation:** Ensuring `os.makedirs()` works correctly on the FUSE mount.
- **Binary file I/O:** Testing the ability to write binary files, such as PNG images generated by Graphviz.

This setup is useful for identifying issues related to pathing, permissions, and file encoding that can occur with FUSE layers in notebook environments.

## File Structure

```
.
├── data1.json
├── data2.json
├── data_science_notebook.ipynb
├── dep1.py
├── dep2.py
└── outputdirectory/
    ├── output1.json
    └── output2.json
```

- **`data_science_notebook.ipynb`**: The main notebook that performs a mock ETL task. It reads from `data1.json` and `data2.json`, uses `dep1.py` and `dep2.py` for transformations and writing, and saves results (`session_stats.json`, `page_views.json`, `etl_pipeline.png`) into `outputdirectory/`.
- **`dep1.py`**: Contains a `DataTransformer` class with methods for processing event data using `pandas`.
- **`dep2.py`**: Contains a `safe_write_json` utility function that ensures an output directory exists before writing JSON data to it.
- **`data1.json` & `data2.json`**: Sample raw event data for the notebook to process.
- **`outputdirectory/`**: A directory intended for storing output files generated by the notebook. It contains placeholder `output1.json` and `output2.json` files and will be populated with `session_stats.json`, `page_views.json`, and `etl_pipeline.png` when the notebook is run.

## How to Use for Testing

1.  Mount this repository using your FUSE implementation (e.g., to `/content/mount/my-repo`).
2.  Navigate to the mounted directory: `cd /content/mount/my-repo`.
3.  Open and run `data_science_notebook.ipynb` in a notebook environment (like Colab or Jupyter).
4.  Verify that the notebook executes successfully, all cells run without `FileNotFoundError` or `ModuleNotFoundError`, and the `outputdirectory/` contains the generated `session_stats.json`, `page_views.json`, and `etl_pipeline.png` files.
